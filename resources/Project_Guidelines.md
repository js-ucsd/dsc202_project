# Project Guidelines

The final project is to create an application that includes:
- One graph data element
- One relational database element
- One vector database element
- IMPORTANT: The need of every store needs to be justified. Do not use them for the sake of it.

## Expectations:
- We want you to frame this project from the perspective of showcasing what your tool does and why.
- We want you to seamlessly include clear explanations of the database structures you choose and why, and how they fit into the purpose of the tool at large.
- You should showcase use cases for your tool/application and how that would utilize the databases.
- Please include visualizations of the schemas of your databases.

## Data Stores to be used:
- PostgreSQL - Relational
- Neo4j - Graph
- Qdrant - Vector

You will be presenting your project to the class. This will be a 15 minute presentation of your project followed by 5 minutes of Q&A. You are encouraged to do a (scripted) live demo of the tool during the presentation.

## Deliverables to be submitted:
- Slides
- Written report that goes into detail about the project
    - why decisions were made the way they were
    - how the work was completed
- Github repo that contains all the code
